# 30条活动种子导入与自动打标验收（逐步点击版）

## 1. 导入前准备

1. 打开微信云开发控制台，确认环境是你当前在用的环境（如 `cloud1-...`）。
2. 进入 `数据库`，找到集合 `activities`。
3. 建议先做一次备份：
   - 点 `导出`，保存一份当前 `activities` 数据（防止误操作）。
4. 准备本次导入文件：
   - [/Users/a1-6/Desktop/小程序/dali-ng-main/daling-mvp/docs/activities_seed_ops_keyword_30.jsonl](/Users/a1-6/Desktop/小程序/dali-ng-main/daling-mvp/docs/activities_seed_ops_keyword_30.jsonl)

## 2. 导入 30 条测试活动（JSON Lines）

1. 在 `数据库 -> activities` 页面，点击 `导入`。
2. 选择文件：
   - `activities_seed_ops_keyword_30.jsonl`
3. 导入格式选择：
   - `JSON`
   - 并确认是 `JSON Lines`（每行一个 JSON 对象）。
4. 导入策略：
   - 建议选 `新增`（不要覆盖现有真实活动）。
5. 点击 `开始导入`，等待任务结束。
6. 成功标准：
   - 任务状态显示成功。
   - `activities` 里可检索到示例标题，如：
   - `三月街早市同游`
   - `微醺小酒馆社交局`
   - `火把节篝火徒步预告`

## 3. 为什么导入后“自动打标”可能看不到

这 30 条是“数据库直导入”，不会自动触发 `publishActivity/updateActivity` 的打标逻辑。  
自动打标是在“发布活动”或“编辑保存活动”时生成的。

所以验收分两段：

1. 用这 30 条先验收“搜索/筛选/场景覆盖/关键词覆盖”。
2. 再做 8 条“自动打标触发验收”（通过发布页真实触发）。

## 5. 自动打标功能验收（真实触发）

下面 8 组，只需你用发布页各发 1 条即可。  
每发完 1 条，去管理后台点进该活动详情，查看“后台标签（结构化自动）”。

### Case A：关键词“三月街”
- 标题：`三月街赶集同游测试A`
- 简介：`三月街在地体验，轻松逛吃`
- 预期：关键词命中含 `云南民俗/非遗主题`。

### Case B：关键词“火把节+篝火”
- 标题：`火把节篝火夜聊测试B`
- 简介：`火把节预热，篝火活动，注意安全`
- 预期：命中 `云南民俗/非遗主题` + `明火风险`。

### Case C：关键词“白族+非遗”
- 标题：`白族非遗扎染体验测试C`
- 简介：`白族非遗手作体验课`
- 预期：命中 `云南民俗/非遗主题`。

### Case D：关键词“交友”
- 标题：`交友破冰桌游测试D`
- 简介：`交友活动，单身可来`
- 预期：命中 `交友敏感`。

### Case E：关键词“微醺”
- 标题：`微醺社交酒局测试E`
- 简介：`微醺小聚，理性饮酒`
- 预期：命中 `酒精相关风险`。

### Case F：关键词“徒步”
- 标题：`苍山徒步拉练测试F`
- 简介：`徒步登山，补给自备`
- 预期：命中 `户外运动风险`。

### Case G：布尔触发器（儿童+宠物）
- 场景：`亲子宠物`
- 类型：`遛狗局` 或 `亲子活动`
- 标题：`亲子宠物同行测试G`
- 预期：布尔风险触发器中 `是否儿童=是`（亲子活动时）或 `是否宠物=是`（遛狗局时）。

### Case H：布尔触发器（审核制）
- 任意活动，发布时开启：`需要发起人审核`
- 标题：`审核制测试H`
- 预期：布尔风险触发器里 `是否审核制=是`。

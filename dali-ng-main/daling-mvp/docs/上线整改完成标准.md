# 搭里上线整改完成标准

## 本次目标

把活动线收回到主 PRD 的首战边界：只做活动信息发布、展示、报名、治理和指标观察，不做小程序内交易、收费活动、退款或收款主体能力。

## 完成定义

- 活动发布页不再出现收费方式、付费金额、收款主体、退款规则入口。
- 活动发布云函数不接受 `aa`、`paid` 或大于 0 的活动费用。
- 活动详情页不展示收费、收款主体、退款规则，不再提示用户核验收费信息。
- 活动协议和发布规则不再把收费活动作为可发布能力描述。
- 管理后台明确展示主 PRD 的活动线首战 5 个指标。
- 租房线保留支付与解锁能力，但房源详情页必须有用户侧投诉入口，解锁前必须有确认说明。
- 重新执行上线检查后，活动线收费冲突不再是 P0 问题。

## 检查标准

1. 在活动线项目中搜索收费、付费、退款、收款主体等词，不能再命中活动发布主流程、活动详情主流程或活动发布云函数的可用能力。
2. 发布云函数语法检查通过，且非免费参数会被拒绝。
3. 管理后台能看到“首战指标”区域，包含：
   - 周发布活动数
   - 活动详情页点击率
   - 报名转化率
   - 成团率
   - 用户次日留存率
4. 租房详情页能看到“投诉/举报房源”入口，点击后会记录投诉事件。
5. 租房支付页保留“解锁前请确认”说明。
6. 更新后的上线检查报告结论不再把活动线收费冲突列为 P0。

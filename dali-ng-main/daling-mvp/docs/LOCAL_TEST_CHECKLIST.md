# 搭里本地回归测试清单

> 适用范围：`daling-mvp` 小程序（uni-app + 微信云开发）
> 目标：每次改动后，快速确认核心链路、合规能力和真机交互可用

## 一、准备工作

1. 在微信开发者工具导入根目录项目：`dali-ng-main/daling-mvp`
2. 确认项目配置：
   - `miniprogramRoot = unpackage/dist/dev/mp-weixin/`
   - `cloudfunctionRoot = cloudfunctions/`
3. 上传并部署云函数（至少本轮相关）：
   - `login`
   - `getActivityDetail`
   - `getActivityList`
   - `publishActivity`
   - `updateActivity`
   - `joinActivity`
   - `quitActivity`
   - `addActivityComment`
   - `getActivityComments`
   - `cancelActivity`
   - `submitReport`
   - `adminAction`
   - `autoUpdateStatus`
   - `checkAdmin`
   - `getAdminDashboard`
   - `startOfficialVerify`
   - `retryOfficialVerify`
   - `officialVerifyCallback`
4. 云函数环境变量确认：
   - `ADMIN_OPENIDS`
   - `TMPL_START`
   - `TMPL_CANCEL`
   - `TMPL_FORMING`
5. 真机预览前确认：
   - 微信公众平台“隐私保护指引”已发布并包含：用户信息、位置信息、选择的位置信息、手机号、剪贴板
   - 涉及真机能力时，使用最新预览二维码重新扫码

## 二、核心冒烟（必须通过）

1. 启动与品牌
   - 启动后首页标题显示“搭里”
   - 协议页、隐私页品牌名为“搭里”
2. 首页活动加载
   - 已授权位置：能加载附近活动
   - 未授权位置：能看到默认区域活动或示例活动
   - 可按“关键词/分类/距离”筛选活动，列表结果正确变化
   - 活动卡片能看到可信度星级与身份标签（A/B/C 对应 5/4/3 星）
3. 登录链路
   - 新用户首次进入不报错
   - 我的页显示默认昵称“搭里用户”
4. 发布活动
   - 未实名：发布被拦截并提示去认证
   - 已实名：可成功发布活动
   - 可选择活动分类（运动/音乐/读书/游戏等）
5. 编辑活动
   - 发布者可进入“编辑活动”页面
   - 可修改地点、开始时间、时长、人数上限
   - 保存后详情页数据同步更新
6. 报名活动
   - 非发布者可报名成功
   - 再次报名会提示“你已经报名了”
   - 发布者不能报名自己的活动
7. 详情页报名状态
   - A 用户报名后，A 显示“已报名”
   - B 用户打开同活动，仍显示可报名（未被误判）
   - A 用户点击“取消报名”后，人数 -1 且按钮恢复“我要参与”
   - 详情页能显示可信度星级、身份标签、风险标签（如“新入驻/临时组织”）
8. 发布者视角参与者列表
   - 发布者进入活动详情可看到参与者列表
   - 列表包含头像、昵称、报名时间
   - 可按昵称关键词筛选参与者
   - 可切换“按最新报名 / 按最早报名”
   - 点击“导出昵称”后可成功复制参与者昵称清单
9. 取消活动
   - 仅发布者可取消
   - 取消后活动状态变为 `CANCELLED`
10. 举报闭环
   - 用户举报后，管理员后台能看到“待处理”
   - 管理员“下架并处理”后，记录显示已处理
   - 管理员“忽略并处理”后，记录显示已忽略
11. 管理后台权限
   - 非管理员进入管理页提示“无管理员权限”
   - 管理员能正常看到审核/举报/活动数据
12. 留言与回复
   - 报名用户在活动详情可发送留言
   - 发布者可点击留言“回复”并发送
   - 未报名用户不可留言（提示“报名后可留言互动”）
13. 活动分享
   - 活动详情底部“分享”按钮可正常唤起微信分享
   - 顶部菜单可分享到朋友圈（标题和落地页正确）
14. 协议与隐私页
   - “用户协议”“隐私保护指引”页面可正常打开
   - 页面内容与当前产品能力一致（含实名认证、位置、剪贴板说明）
15. 自动状态更新
   - `autoUpdateStatus` 云函数可手动调用成功
   - 过期活动会更新为 `ENDED`
   - 成团活动在截止后会更新正确的 `formationStatus`
16. 管理报表导出（二期+趋势）
   - “导出基础报表 CSV”可成功导出
   - “导出报表二期 CSV（双表）”可生成 Summary + Detail 两份文件
   - “导出周/月趋势 CSV”可成功导出
   - 趋势 CSV 中可看到周维度、月维度、环比字段
17. 官方实名 3.1（生产收口）
   - 回调签名正确时返回 `success: true`
   - 重复回调不重复落库，返回幂等成功
   - 构造错误签名时返回 `SIGNATURE_MISMATCH`
   - 同 nonce 的疑似重放请求会被拦截（`FAILED_REPLAY_ATTACK`）
   - 管理后台“官方实名审计看板”可看到即时告警/阈值告警/重放拦截指标

## 三、合规检查（快速版）

1. 小程序页面文案不出现：
   - 支付、价格、订单、会员费、充值
2. 不出现小程序内支付路径或支付按钮
3. 发布前仍需实名认证
4. 用户协议和隐私页文案与后台隐私保护指引口径一致
5. 真机调用位置、剪贴板等能力时，不出现“未在隐私保护指引声明”报错

## 四、异常场景（建议）

1. 断网后进入首页，页面有可理解反馈
2. 云函数异常时，前端有 toast 提示，不白屏
3. 手机授权拒绝后，可继续浏览，不阻塞
4. 订阅消息未授权时，报名流程仍可继续，不阻塞主链路
5. 剪贴板复制失败时，页面有明确提示，不出现无响应

## 五、通过标准

1. “核心冒烟” 17 项全部通过
2. “合规检查” 5 项全部通过
3. 无阻断性报错（白屏、无法进入核心页、云函数主链路失败）

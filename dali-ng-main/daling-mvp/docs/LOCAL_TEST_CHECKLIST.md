# 搭里本地回归测试清单（MVP）

> 适用范围：`daling-mvp` 小程序（uni-app + 微信云开发）
> 目标：每次改动后，快速确认核心链路可用

## 一、准备工作

1. 在微信开发者工具导入根目录项目：`dali-ng-main/daling-mvp`
2. 确认项目配置：
   - `miniprogramRoot = unpackage/dist/dev/mp-weixin/`
   - `cloudfunctionRoot = cloudfunctions/`
3. 上传并部署云函数（至少本轮相关）：
   - `login`
   - `getActivityDetail`
   - `getActivityList`
   - `publishActivity`
   - `joinActivity`
   - `quitActivity`
   - `addActivityComment`
   - `getActivityComments`
   - `cancelActivity`
   - `submitReport`
   - `adminAction`
   - `autoUpdateStatus`
   - `checkAdmin`
   - `getAdminDashboard`
4. 云函数环境变量确认：
   - `ADMIN_OPENIDS`
   - `TMPL_START`
   - `TMPL_CANCEL`
   - `TMPL_FORMING`

## 二、核心冒烟（必须通过）

1. 启动与品牌
   - 启动后首页标题显示“搭里”
   - 协议页、隐私页品牌名为“搭里”
2. 首页活动加载
   - 已授权位置：能加载附近活动
   - 未授权位置：能看到默认区域活动或示例活动
   - 可按“关键词/分类/距离”筛选活动，列表结果正确变化
   - 活动卡片能看到可信度星级与身份标签（A/B/C 对应 5/4/3 星）
3. 登录链路
   - 新用户首次进入不报错
   - 我的页显示默认昵称“搭里用户”
4. 发布活动
   - 未实名：发布被拦截并提示去认证
   - 已实名：可成功发布活动
   - 可选择活动分类（运动/音乐/读书/游戏等）
5. 报名活动
   - 非发布者可报名成功
   - 再次报名会提示“你已经报名了”
   - 发布者不能报名自己的活动
6. 详情页报名状态
   - A 用户报名后，A 显示“已报名”
   - B 用户打开同活动，仍显示可报名（未被误判）
   - A 用户点击“取消报名”后，人数 -1 且按钮恢复“我要参与”
   - 详情页能显示可信度星级、身份标签、风险标签（如“新入驻/临时组织”）
7. 发布者视角参与者列表
   - 发布者进入活动详情可看到参与者列表
   - 列表包含头像、昵称、报名时间
8. 取消活动
   - 仅发布者可取消
   - 取消后活动状态变为 `CANCELLED`
9. 举报闭环
   - 用户举报后，管理员后台能看到“待处理”
   - 管理员“下架并处理”后，记录显示已处理
   - 管理员“忽略并处理”后，记录显示已忽略
10. 管理后台权限
   - 非管理员进入管理页提示“无管理员权限”
   - 管理员能正常看到审核/举报/活动数据
11. 留言与回复
   - 报名用户在活动详情可发送留言
   - 发布者可点击留言“回复”并发送
   - 未报名用户不可留言（提示“报名后可留言互动”）
12. 活动分享
   - 活动详情底部“分享”按钮可正常唤起微信分享
   - 顶部菜单可分享到朋友圈（标题和落地页正确）

## 三、合规检查（快速版）

1. 小程序页面文案不出现：
   - 支付、价格、订单、会员费、充值
2. 不出现小程序内支付路径或支付按钮
3. 发布前仍需实名认证
4. 用户协议和隐私页可正常访问

## 四、异常场景（建议）

1. 断网后进入首页，页面有可理解反馈
2. 云函数异常时，前端有 toast 提示，不白屏
3. 手机授权拒绝后，可继续浏览，不阻塞

## 五、通过标准

1. “核心冒烟” 12 项全部通过
2. “合规检查” 4 项全部通过
3. 无阻断性报错（白屏、无法进入核心页、云函数主链路失败）

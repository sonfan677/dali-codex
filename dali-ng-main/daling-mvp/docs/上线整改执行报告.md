# 搭里上线整改执行报告

## 结论

本轮整改已把活动线收费冲突从 P0 降为已处理：活动发布主流程不再提供收费、收款主体、退款规则入口，发布云函数会拒绝非免费活动参数。

租房线复查已完成：房源详情页已补用户侧投诉/举报入口；租房支付页保留“解锁前请确认”。

## 已完成

- 活动发布页移除收费方式、付费金额、收款主体、退款规则入口。
- 活动发布页保存草稿、套用模板、恢复草稿时都强制回到免费活动。
- `publishActivity` 云函数拒绝 `aa`、`paid` 或大于 0 的活动费用。
- 活动详情页不再展示费用、收款主体、退款规则。
- 活动报名风险提示不再因为收费进入确认项。
- 活动发布规则、组织者协议、用户协议改为活动线不提供收费、收款、退款、担保能力。
- 活动后台新增“首战指标”区域，固定展示主 PRD 的 5 个活动线指标。
- 租房详情页新增“投诉/举报房源”入口。
- 租房工具层新增房源投诉记录，并同步增加房源投诉数。

## 首战 5 个指标

- 周发布活动数
- 活动详情页点击率
- 报名转化率
- 成团率
- 用户次日留存率

说明：当前“活动详情页点击率”和依赖详情访问量的“报名转化率”需要详情访问埋点；没有埋点数据时后台显示“待埋点”，不伪造成真实结果。

## 复查结果

- 活动发布页、活动详情页、发布云函数、详情云函数、报名云函数、编辑云函数中，未再发现收费入口或收款/退款入口。
- `publishActivity` 已有 `ACTIVITY_CHARGE_NOT_SUPPORTED` 拦截。
- 后台已有 `launchMetricCards`，并在管理后台顶部展示“首战指标”。
- 租房详情页已有“投诉/举报房源”按钮。
- 租房支付页已有“解锁前请确认”。

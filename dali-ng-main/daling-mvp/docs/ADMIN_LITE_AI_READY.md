# 搭里后台瘦身 1.0（人工高效 + AI 预留）

## 目标
- 人工管理优先：默认进入「精简模式」，减少干扰信息，保留高频动作。
- AI 渐进接入：先补齐动作日志字段和 dry-run 预检查，不改变当前人工流程。

## 本轮改动

### 1) 后台精简模式
- 顶部新增「精简模式 / 完整模式」切换。
- 精简模式下默认收起：
  - 日报归档高级区
  - SLA 细分看板与闭环明细趋势
  - 操作记录页的导出/趋势高级分析
- 完整模式下保留全部能力。

### 2) 运营日报自动归档（人工可用）
- 新增每日自动归档开关（本地持久化）。
- 每天首次进入后台（且有权限）自动归档一条日报快照。
- 提供：
  - 复制最近归档
  - 按日期复制归档日报

### 3) SLA 细分看板（可折叠）
- 新增三个分组的时长分布：
  - 举报待办等待时长
  - 认证待办等待时长
  - 已闭环处理时长
- 桶区间：`<6h / 6-12h / 12-24h / >=24h`。

### 4) AI 对接预留（云函数）
- `adminAction` 新增接收并落日志字段：
  - `actionSource`（`human`/`ai`）
  - `canAutoExecute`（布尔）
  - `manualOverride`（布尔）
  - `dryRun`（布尔）
  - `agentTraceId`（字符串）
- `dryRun=true` 时返回预检查成功，不执行真实写操作。
- `getAdminDashboard` 已回传以上字段，前端日志页可显示来源（人工/AI）。

## 部署提醒
- 本轮改动包含云函数逻辑，需重新上传：
  - `cloudfunctions/adminAction`
  - `cloudfunctions/getAdminDashboard`

## 建议验收
1. 精简/完整模式切换是否生效。
2. 日报自动归档开关与“复制最近归档”是否可用。
3. 操作记录里是否显示执行来源（人工/AI）。
4. `dryRun` 预检查（控制台调用）是否返回 `dryRun: true` 且不改数据。
